# Submission Requirements

Built something awesome? Great.

Before your project can earn currency, get reviewed, and start its journey through the Alchemize cauldron, make sure it follows the requirements below.

Think of this as a pre-flight checklist for your project.

---

## Track Your Time

All project work must be tracked using:

* Lapse
* Hackatime

No tracked time = no currency.

The tracked project should correspond to the project you're submitting. If your Hackatime says you're building a calculator while your submission is a 3D game engine, reviewers may have questions.

---

## Open Source Everything

Your code must be publicly available.

Supported platforms:

* GitHub
* GitLab

If reviewers can't see the code, they can't review the project.

Closed-source submissions are not eligible.

---

## README Requirements

Every project repository must contain a README.

Not just a README that says:

> cool project lol

A proper README should explain:

### What is the project?

Tell us what you built.

### Why did you build it?

What problem does it solve? What made you want to create it?

### How do I test it?

A reviewer should be able to follow your instructions and try the project themselves.

### Inspiration

What sparked the idea?

Maybe it was a game you loved, a problem you faced, or a random shower thought at 2 AM.

### Theme

This is important.

Include a dedicated **Theme** section explaining why your project belongs to the theme you selected.

The standard is simple:

A random Hack Clubber should be able to:

1. Read your Theme section.
2. Read the official theme guidelines.
3. Conclude that your project belongs in that theme.

If reviewers need to perform archaeological excavations to figure out why your project fits the theme, rewrite the section.

---



## Demo Link

Reviewers need a way to actually use your project.

Your demo link should be stable and accessible long after you submit.

### Good Examples

#### Web Projects

* GitHub Pages
* Vercel
* Netlify
* Permanent custom hosting

#### Mobile Apps

* Google Play Store
* Apple App Store
* GitHub Releases

#### Desktop Apps

* GitHub Releases containing executables or installers

### Not Accepted

* Cloudflare Tunnels
* ngrok
* localhost links
* Self-hosted machines that might disappear overnight
* Temporary preview deployments
* "It worked yesterday, I promise"

If reviewers can't access your project, they can't review it.

---

## Screenshot Requirement

Every submission must include a cover screenshot.

The screenshot should show your project running.

Examples:

* Your website
* Your game
* Your desktop app
* Your mobile app
* Your terminal application

Please do **not** upload screenshots of:

* VS Code
* Cursor
* Vim
* Neovim
* IntelliJ
* A wall of code
* Your browser displaying GitHub

We want to see what you built, not where you built it.

---

## Project Description

Every submission must include a description that clearly explains the project.

Requirements:

* At least 50 characters long
* Explains what the project does
* Gives reviewers enough context to understand it

A good description tells reviewers what they're about to look at.

A bad description is:

> cool app

---

## Theme Requirement

Choose a theme because your project belongs there, not because the currency name sounds cooler.

Your project should genuinely satisfy the selected theme's requirements.

Reviewers will use:

* Your project
* Your README
* Your demo
* Your Theme section
* The official theme guidelines

to determine eligibility.

If a project doesn't fit the chosen theme, it may be rejected or moved to a more appropriate one.

---
## Demo Video(Optional but Strongly Recommended)

Every project must include a demo video linked in the README.

Requirements:

* Uploaded to YouTube
* Public or Unlisted
* Shows the project actually working
* Screen recording of the project being demonstrated

Please show the project doing project things.

A 5-minute tour of your source code does not count as a demo.

---
## Final Checklist

Before submitting, make sure you have:

* Time tracked with Lapse or Hackatime
* Public GitHub or GitLab repository
* README with all required sections
* Theme justification section
* YouTube demo video
* Working demo link
* Demo screenshot
* Project description (50+ characters)
* A project that genuinely fits the selected theme

If you've checked every box, toss your project into the cauldron and submit away.
